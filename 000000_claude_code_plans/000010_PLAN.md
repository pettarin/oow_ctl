# Plan 010: Set USER_AGENT env var for Open WebUI to silence langchain warning

## Context
The Open WebUI container logs a warning from `langchain_community.utils.user_agent`:
`USER_AGENT environment variable not set, consider setting it to identify your requests.`
The fix is to pass a `USER_AGENT` environment variable to the `svc-open-webui` service.

## Changes

### 1. Add `OPENWEBUI_USER_AGENT` variable to `oow_ctl.sh`

- [x] Add a new default-value export alongside the other Open WebUI variables (after line 17):
```bash
export OPENWEBUI_USER_AGENT="${OPENWEBUI_USER_AGENT:-Mozilla/5.0 (X11; Linux x86_64; rv:149.0) Gecko/20100101 Firefox/149.0}"
```

### 2. Pass `USER_AGENT` to the container in `docker-compose.yml`

- [x] Add `USER_AGENT=${OPENWEBUI_USER_AGENT}` to the `environment` list of `svc-open-webui` (after the existing `WEBUI_AUTH` entry):
```yaml
    environment:
      - OLLAMA_BASE_URL=http://svc-ollama:11434
      - WEBUI_AUTH=false
      - USER_AGENT=${OPENWEBUI_USER_AGENT}
```

## Files modified
- `oow_ctl.sh`
- `docker-compose.yml`

## Verification
- [x] Restart the stack with `./oow_ctl.sh stop && ./oow_ctl.sh start`
- [x] Run `./oow_ctl.sh logs` and confirm the `USER_AGENT environment variable not set` warning is gone
