# Plan 009: Rename COMPOSE_FILE variable and honor pre-existing env vars

## Context
Two housekeeping improvements to `oow_ctl.sh`:
1. The internal variable `COMPOSE_FILE` should be renamed to `DOCKER_COMPOSE_MANIFEST_PATH` for consistency with the existing `DOCKER_COMPOSE_MANIFEST_FILE_NAME` variable.
2. The `export` statements unconditionally overwrite any values already set in the caller's environment. They should be changed to only set defaults, so that a caller can override any variable before invoking the script.

## Changes to `oow_ctl.sh`

### 1. Rename `COMPOSE_FILE` → `DOCKER_COMPOSE_MANIFEST_PATH`

- [x] Line 23: rename the variable assignment from `COMPOSE_FILE` to `DOCKER_COMPOSE_MANIFEST_PATH`
- [x] Lines 45, 49, 53, 58, 63: replace every `"$COMPOSE_FILE"` with `"$DOCKER_COMPOSE_MANIFEST_PATH"`

### 2. Honor pre-existing environment variables

- [x] Change each `export VAR="value"` (lines 7–17) to use the `${VAR:-default}` pattern, e.g.:

```bash
export DOCKER_COMPOSE_MANIFEST_FILE_NAME="${DOCKER_COMPOSE_MANIFEST_FILE_NAME:-docker-compose.yml}"
export OLLAMA_CONTAINER_NAME="${OLLAMA_CONTAINER_NAME:-ollama}"
export OLLAMA_IMAGE="${OLLAMA_IMAGE:-ollama/ollama}"
export OLLAMA_IMAGE_TAG="${OLLAMA_IMAGE_TAG:-0.20.0}"
export OLLAMA_PORT="${OLLAMA_PORT:-11434}"
export OLLAMA_VOLUME="${OLLAMA_VOLUME:-/mnt/usb/docker/ollama}"
export OPENWEBUI_CONTAINER_NAME="${OPENWEBUI_CONTAINER_NAME:-open-webui}"
export OPENWEBUI_IMAGE="${OPENWEBUI_IMAGE:-openwebui/open-webui}"
export OPENWEBUI_IMAGE_TAG="${OPENWEBUI_IMAGE_TAG:-latest}"
export OPENWEBUI_PORT="${OPENWEBUI_PORT:-8080}"
export OPENWEBUI_VOLUME="${OPENWEBUI_VOLUME:-/mnt/usb/docker/open-webui}"
```

## Verification
- [x] `grep COMPOSE_FILE oow_ctl.sh` returns no matches (old name fully removed)
- [x] `grep DOCKER_COMPOSE_MANIFEST_PATH oow_ctl.sh` shows the new variable in the assignment and all `docker compose -f` calls
- [x] `./oow_ctl.sh status` works as before (uses defaults)
- [x] `OLLAMA_PORT=9999 ./oow_ctl.sh status` honors the override (confirmed Ollama mapped to port 9999)
